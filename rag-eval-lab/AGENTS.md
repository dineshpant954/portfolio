# AGENTS.md

## Role
You are helping maintain this repository as part of Dinesh Pant’s AI Builder / AI Product Manager portfolio.

## Style
- Keep code simple, readable, and demo-friendly.
- Prefer small working examples over over-engineered systems.
- Every feature should connect to a clear user problem.
- Document product thinking, not just implementation.
- Do not use private data, credentials, or real customer information.

## Review guidelines
- Check for broken setup instructions.
- Check that README explains the product value clearly.
- Check that sample data is synthetic.
- Check that claims are realistic and not inflated.
- Flag missing evaluation logic for AI features.
- Flag security/privacy risks.
