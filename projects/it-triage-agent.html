// Project 003

IT Support Ticket
Triage & Resolution

Agentic AI · IT Ops · Project 003

Enterprise IT service desks process 10,000–100,000+ tickets/month. Tier-1 costs ~$22/ticket; escalations to Tier-3 exceed $104/ticket. 40–60% of volume is repetitive — password resets, VPN issues, software requests — yet still requires a human to read, classify, route, and resolve. This agent changes that: it classifies intent, retrieves relevant knowledge, executes safe resolutions autonomously, and escalates with full context when it can't.

// 03 · Architecture

How it's built

Intake Layer
Slack Bot, Teams Bot, Email, Portal → n8n webhooks normalize every channel into a unified ticket schema
Triage Agent
Claude API structured output → intent classification + priority scoring + confidence thresholds in a single call
Knowledge Retrieval
Supabase pgvector + Confluence KB → RAG with citations, retrieves relevant articles and past resolutions
Resolution Engine
Mock APIs (Azure AD, Intune, Jamf) → tool execution with rollback support and audit logging
Policy & Approval
Intent whitelist + Slack approval flow (n8n Wait node) → policy-gated autonomy per intent category
Observability
Execution logs + audit trail + Grafana dashboard → full traceability from intake to resolution
