// Project 003

IT Support Ticket Triage
& Resolution Agent

Agentic AI system that classifies, retrieves, and resolves IT tickets autonomously
n8n Groq (Llama 3.3 70B) Supabase Slack

Enterprise IT service desks process 10,000-100,000+ tickets per month. 40-60% of that volume is repetitive - password resets, VPN issues, software requests. This agent classifies tickets by intent and priority, searches a knowledge base for relevant articles, checks policy rules to determine the right level of autonomy, and either auto-resolves, routes for human approval, or escalates with full context - all through Slack.

Security-First
Least privilege for every tool call. Every action audited. PII masked.
Human Trust Ramp
Copilot to HITL to lights-out. Autonomy earned through measured performance.
Environment-Agnostic
Adapter pattern for ServiceNow, JSM, Zendesk. Works with what exists.
Measurable ROI
Every capability tied to a metric. Deflection rate, MTTR, cost per ticket.

// 03 - Architecture

How it's built

Intake
Slack #it-support - n8n webhook receives messages, normalizes into unified ticket schema
Triage
Groq / Llama 3.3 70B - classifies intent, priority, confidence via structured JSON output
Retrieval
Supabase full-text search - matches ticket against 20 KB articles, returns top 3 with relevance scores
Policy Check
Intent whitelist + confidence threshold - determines L3 (auto), L2 (approval), or escalation
Resolution
Mock tool APIs (Azure AD, Okta, SCCM) - execute actions or post approval requests to Slack
Audit Log
Supabase Postgres - every agent action logged with timestamp, tool, parameters, result
Groq
$0 cost, sub-second inference, Llama 3.3 70B with tool calling support. OpenAI-compatible API works with n8n's native Groq node.
n8n
Visual orchestration that's screenshot-able for a portfolio. Native Groq node, 400+ integrations, built-in Slack and HTTP nodes. Free self-hosted.
Supabase
Free Postgres with pgvector and full-text search. REST API built-in. Handles KB articles, tickets, and audit logs in one database.
Slack
Real-time user interface for ticket intake. Block Kit for approval buttons in HITL workflows. Native n8n integration.
What's Real vs Mocked
Real: LLM reasoning (Groq/Llama), workflow orchestration (n8n), knowledge base search (Supabase), Slack integration, audit logging, policy routing.
Mocked: Tool execution (Azure AD, Okta, SCCM) uses an Express server returning realistic payloads. In production, each mock endpoint maps to a real enterprise API call. All mock responses are transparently labeled.
